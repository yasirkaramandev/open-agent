# Signal — YouTube Operations Console

Signal is a safety-first reference architecture for teams that monitor YouTube channels and manage
their own channel operations. Monitoring may run automatically. Engagement never does: every write
is a reviewable proposal bound to one account, an exact payload hash, an approval, an idempotency
key, and an append-only audit event.

This example intentionally ships with fake adapters and feature-gated write execution. It never asks
for a Google password, imports browser sessions, automates Google login, rotates accounts, or offers
bulk engagement.

## Repository map

- `apps/web` — responsive Next.js operations console and approval experience.
- `apps/api` — FastAPI contract, OAuth safety primitives, proposal state machine, RBAC, and models.
- `apps/worker` — monitoring-only task allowlist and write execution guard.
- `packages` — API contract, UI tokens, routing schema, and shared types.
- `infra/docker` — non-root local PostgreSQL/Redis/API composition.
- `tests/security` — repository policy and prohibited-field regression checks.
- `docs` — architecture, OAuth threat model, deployment, retention, and incident notes.

## Local development

The web demo uses realistic local data and does not need credentials:

```bash
cd apps/web
npm install
npm run dev
```

The API is a contract reference:

```bash
cd apps/api
python -m venv .venv
.venv/bin/pip install -e ".[dev]"
.venv/bin/pytest
.venv/bin/uvicorn signal_api.main:app --reload
```

For real OAuth, create a Google Cloud OAuth client with a fixed HTTPS callback. Request read-only
YouTube scope first. Write scopes are incremental and requested only while the user initiates a
specific comment, rating, or subscription proposal. Set production secrets in the deployment secret
manager, never in this repository.

compliance when applicable → orchestrator integration.

## Status

Foundation, interaction prototype, domain contracts, fake monitoring flow, security controls, and
tests are implemented. Real Google OAuth exchange, KMS, PostgreSQL/Redis deployment, and YouTube API
writes remain intentionally disabled until operator credentials and infrastructure are supplied.
